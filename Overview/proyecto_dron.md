# Project Overview – Sistema de Dron Basado en Kit

## 1. Descripción del Proyecto

El presente proyecto consiste en el diseño e integración de un sistema de dron utilizando componentes comerciales previamente fabricados, en lugar de construir la aeronave desde cero. La propuesta se centra en la adquisición de un kit de dron junto con el desarrollo de un sistema de control personalizado.

El dron será ensamblado a partir de un kit que incluye la estructura, motores y controladores básicos. Paralelamente, se diseñará un control remoto propio mediante la integración de joysticks, un módulo de comunicación LoRa, baterías y otros componentes electrónicos, los cuales serán montados en una PCB (placa de circuito impreso).

El objetivo del proyecto no es fabricar cada componente individual, sino integrar tecnologías existentes para obtener un sistema funcional, reduciendo costos, tiempos de desarrollo y complejidad técnica.

---

## 2. Objetivo del Proyecto

Desarrollar un sistema de dron funcional mediante la integración de un kit comercial y un control remoto personalizado, capaz de establecer comunicación inalámbrica confiable utilizando tecnología LoRa.

### Objetivos específicos

- Ensamblar correctamente el dron a partir del kit adquirido.
- Diseñar e implementar un control remoto basado en joysticks.
- Integrar comunicación inalámbrica mediante módulos LoRa.
- Diseñar una PCB para el sistema de control.
- Garantizar operación estable, segura y eficiente del dron.

---

## 3. Factibilidad del Proyecto

### Factibilidad Técnica

El proyecto es técnicamente viable debido a que:

- Se emplean kits comerciales previamente probados.
- La tecnología LoRa es ampliamente utilizada y documentada.
- Existen herramientas accesibles para el diseño de PCB.
- No se requiere fabricación compleja de hardware crítico (motores, frame, etc.).

### Factibilidad Económica

El costo del proyecto se reduce significativamente al:

- Evitar el desarrollo completo del dron desde cero.
- Utilizar componentes comerciales disponibles en el mercado.
- Minimizar riesgos asociados a fallos de diseño mecánico.

### Factibilidad Operativa

El sistema puede ser operado por usuarios con conocimientos básicos en electrónica y sistemas embebidos, siempre que se sigan los procedimientos de ensamblaje y configuración.

### Riesgos Identificados

- Posible incompatibilidad entre módulos electrónicos.
- Problemas en la comunicación inalámbrica.
- Errores en el diseño de la PCB.
- Limitaciones energéticas o interferencias.
